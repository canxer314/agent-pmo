# /acceptance — 验收管理

创建阶段验收报告和终验报告，跟踪不符合项，确保验收标准与合同对齐。

## 定位

| Skill | 职责 | 输出 |
|-------|------|------|
| `/plan` | 计划制定 | 里程碑定义（含验收标准） |
| `/meeting` | 会议纪要 | 评审会记录、验收问题识别 |
| `/acceptance` | **验收管理 — 阶段验收 + 终验 + 不符合项跟踪** | 验收报告 + 不符合项清单 |
| `/change` | 变更管理 | 验收不通过触发的变更 |
| `/close` | 项目收尾 | 决算 + 复盘 |

**典型工作流**: 里程碑完成 → `/meeting type=review` 评审 → `/acceptance action=stage` 阶段验收 → 不符合项整改 → `/acceptance action=final` 终验 → `/close` 收尾

---

## Behavior

### Step 0: 确定动作

| action | 行为 |
|--------|------|
| 默认（无参数）| 交互式选择：阶段验收 / 终验 |
| `stage` | 创建阶段验收报告 |
| `final` | 创建终验报告 |

---

### Step 1: 选择项目

如提供 `project` 参数：

```bash
obsidian read path="项目库/{project}/00-项目章程.md"
```

未提供时：

```bash
obsidian search query="type/project"
```

过滤 `status: executing` 或 `status: monitoring` 的项目供选择。

---

### Step 2: 读取验收上下文

```bash
obsidian read path="项目库/{project}/01-合同/主合同关键条款.md"
obsidian read path="项目库/{project}/02-计划/里程碑计划.md"
```

提取关键约束：
- 合同验收条件（验收标准、验收流程、交付物清单）
- 里程碑完成标准（如 action=stage）

---

### Step 3: 阶段验收（action=stage）

#### 3a. 选择里程碑

如提供 `milestone` 参数，直接使用。否则列出项目里程碑供选择：

```bash
obsidian read path="项目库/{project}/02-计划/里程碑计划.md"
```

过滤 `status: in-progress` 或 `status: done` 的里程碑（not-started 的不应验收）。

#### 3b. 读取里程碑详情

```bash
obsidian read path="项目库/{project}/02-计划/里程碑-{名称}.md"
```

提取里程碑的完成标准和 deliverable 清单。

#### 3c. 收集验收信息

与用户对话收集：

| 字段 | 说明 |
|------|------|
| 验收日期 | 默认今天 |
| 验收方式 | 现场验收 / 远程验收 / 文档评审 |
| 验收参与人 | 客户方 + 我方 |
| 验收依据 | 合同验收条件 + 里程碑完成标准 |
| 交付物检查 | 逐项确认里程碑 deliverable 是否已提交且评审通过 |
| 不符合项 | 逐条记录：描述、严重度、整改责任人、整改期限 |

## 与其他 Skill 的协作

| 场景 | 组合 |
|------|------|
| 里程碑完成准备验收 | `/monitor` 确认进度 → `/acceptance action=stage` |
| 评审会发现验收问题 | `/meeting type=review` → `/acceptance action=stage` |
| 验收不通过需变更 | `/acceptance` → `/change` |
| 终验通过后收尾 | `/acceptance action=final` → `/close` |
| 查询历史验收记录 | `/query "项目X的验收情况"` |

---

## Notes

- `/acceptance` 覆盖从阶段验收到终验的完整流程，填补执行与收尾之间的真空
- 阶段验收关联具体里程碑，一个里程碑对应一次阶段验收
- 终验汇总全部阶段验收结果，是进入 `/close` 的前置条件
- 不符合项（NC）必须在验收报告中记录，整改后回写到报告
- 终验通过的验收结论具有合同法律效力，签字确认不可省略
