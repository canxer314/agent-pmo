# /contract — 合同管理

审查合同条款、管理签订与补充协议、跟踪履约进度。承接 `/initiate` 创建的合同骨架，覆盖合同签订→履行→变更→归档全周期。

## 定位

| Skill | 职责 | 输出 |
|-------|------|------|
| `/initiate` | 项目立项 — 创建合同关键条款骨架 | `01-合同/主合同关键条款.md` |
| `/contract` | **合同管理 — 审查+签订+补充协议+履约跟踪** | 履行清单 + 签订记录 + 补充协议 |
| `/payment` | 回款跟踪 — 按付款节点催款 | 回款汇总视图 |
| `/change` | 变更管理 — 范围/工期/成本变更 | 变更记录 CR |
| `/close` | 项目收尾 — 决算+复盘 | 决算报告 |

**典型工作流**: `/initiate` 立项 → `/contract action=review` 审查 → `/contract action=sign` 签订 → `/contract action=track`（定期）→ `/close` 收尾

---

### 合同状态推导

`/contract` 负责合同生命周期状态推导。AI 不静默设置状态，每次操作后通过双提议建议下一状态，用户确认或纠正。

| 状态 | 含义 | 触发条件 |
|------|------|----------|
| `draft` | 合同骨架已创建 | `/initiate` 创建立项后 |
| `reviewed` | 条款已审查，履行清单已生成 | `/contract action=review` 完成后 |
| `signed` | 已签订 | `/contract action=sign` 完成后 |
| `executing` | 履行中 | 项目进入 executing 状态且合同已签订 |
| `amended` | 有补充协议 | `/contract action=amend` 完成后 |
| `fulfilled` | 全部履行完毕 | 履行义务清单全部 done + 回款全部完成 |
| `breached` | 出现违约 | 义务清单有逾期未完成项 |

**推导方法**：每次操作前，读取 `01-合同/` 目录下已有文件判断当前状态；操作完成后在双提议中建议下一状态。

| action | 行为 |
|--------|------|
| `review`（默认）| 审查合同关键条款，生成履行义务清单 |
| `sign` | 记录合同签订状态和原件归档位置 |
| `amend` | 管理补充协议（创建/查看） |
| `track` | 履约跟踪 — 检查义务清单 + 预警 |

## 与其他 Skill 的协作

| 场景 | 组合 |
|------|------|
| 立项后审查合同 | `/initiate` → `/contract action=review` |
| 合同签订记录 | `/contract action=review` → `/contract action=sign` |
| 变更触发补充协议 | `/change` → `/contract action=amend` |
| 定期履约检查 | `/contract action=track` → `/payment`（如涉及回款） |
| 收尾前最终履约确认 | `/contract action=track` → `/close` |
| 回款节点核对 | `/payment` → `/contract action=track` |

---

## Notes

- `/contract` 承接 `/initiate` 创建的合同骨架，覆盖签订→履行→变更→归档全周期
- 履行义务清单是合同执行的 checklist，应与里程碑计划和回款跟踪交叉核对
- 补充协议编号 `SA-{NNN}` 是项目内顺序号，从 001 开始
- 合同履约状态不影响项目状态，但 `/close` 前必须确认合同 fulfilled
- `/contract action=track` 建议每月至少运行一次
- AI 不在无用户确认的情况下修改合同相关条款（合同数据具有法律审计价值）
