# /initiate — 项目立项

中标后创建完整项目结构，是项目生命周期的起点。

## 定位

| Skill | 职责 | 输出 |
|-------|------|------|
| `/bid` | 投标管理 | 投标档案 |
| `/initiate` | **项目立项 — 唯一创建项目结构的通道** | 项目文件夹 + 章程 + 初始文档 |
| `/plan` | 计划制定 | WBS + 里程碑 |

**典型工作流**: `/bid` 中标 → `/initiate` 立项 → `/plan` 制定计划

---

## Behavior

### Step 1: 获取项目信息

#### 1a. 关联投标（如提供 bid 参数）

```bash
obsidian read path="投标档案/投标-{客户}-{主题}-{日期}/投标结果.md"
obsidian read path="投标档案/投标-{客户}-{主题}-{日期}/商务报价.md"
obsidian read path="投标档案/投标-{客户}-{主题}-{日期}/招标要求.md"
obsidian read path="投标档案/投标-{客户}-{主题}-{日期}/解决方案.md"
obsidian read path="投标档案/投标-{客户}-{主题}-{日期}/客户需求分析.md"
```

提取信息：
- 客户、主题、中标金额、预算限价、付款条件
- **方案承诺**（从 解决方案.md 提取）：产品/服务配置 → 写入项目章程「方案承诺摘要」
- **关键需求**（从 客户需求分析.md 提取）：需求优先级清单 → 写入项目章程「需求基线」
- **预判风险**（从 解决方案.md 提取）：风险与应对表 → 预填充到风险登记册（标注"方案阶段预判"）

#### 1b. 无投标档案时

与用户对话收集：

| 字段 | 说明 |
|------|------|
| 项目名称 | 项目全称 |
| 客户名称 | 客户全称 |
| 合同金额 | 签约金额 |
| 项目经理 | 本人（单人使用） |
| 业务经理 | 本人（单人使用） |
| 合同起止日期 | 开始日期、结束日期 |
| 付款节点 | 各阶段付款比例和条件 |
| 验收条件 | 终验和阶段验收条件 |
| 质保条款 | 质保金比例、质保期 |
| 总体预算 | 项目总预算（成本侧） |

#### 1c. 生成分配项目编号

按 SCHEMA.md §11 规则：`PJ-YYYY-NNN`

搜索现有项目确定下一个编号：

```bash
obsidian folders
```

找到 `项目库/` 下最大的编号，+1。

## Notes

- `/initiate` 是 vault **唯一创建项目结构的通道**
- 项目编号一旦分配，不可更改
- 合同关键条款是后续 `/plan`、 `/change`、 `/monitor` 的约束来源
- 预算执行表的初始数据来自投标阶段的成本测算
