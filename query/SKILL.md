# /query — 项目治理查询

带 vault 上下文回答项目治理问题。

## 定位

| Skill | 职责 | 输出 |
|-------|------|------|
| `/query` | **跨项目查询 — 利用项目档案作答** | 带来源标注的回答 |
| `/monitor` | 运营监控 | 单项目健康报告 |
| `/lint` | 项目体检 | 结构健康检查 |

---

## Behavior

### Step 1: 解析查询意图

从用户问题中提取：
- 查询类型：项目健康 / 客户历史 / 投标统计 / 预算分析 / 风险查询 / 知识复用
- 涉及的项目/客户/时间范围
- 输出形式：列表 / 统计 / 对比 / 建议

### Step 2: 搜索 Vault

#### 2a. 读取全局 MOC

```bash
obsidian read path="MOC-Projects.md"
obsidian read path="MOC-Clients.md"
obsidian read path="MOC-Lessons.md"
```

#### 2b. 针对性搜索

| 查询类型 | 搜索策略 |
|----------|----------|
| 项目健康度 | 读项目章程 → 读监控文件 |
| 预算超支 | 扫描所有预算执行表 |
| 客户历史 | 读客户档案 → 搜索关联项目 |
| 投标成功率 | 统计线索池 + 投标档案 |
| 风险查询 | 扫描所有风险登记册 |
| 知识复用 | 搜索知识库/经验教训 |

```bash
obsidian search query="{关键词}"
obsidian search:context query="{关键短语}"
```

### Step 3: 读取命中的文档

```bash
for doc in {搜索结果}; do
  obsidian read path="$doc"
done
```

优先读取：
1. 项目章程 — 综合信息密度最高
2. 预算执行表 — 财务数据
3. 风险登记册 — 风险状态
4. 经验教训 — 可复用知识

### Step 4: 综合回答

回答原则：
- **Vault 知识优先**：vault 中有的信息，标注来源 `[[文档名]]`
- **模型知识补充**：vault 中没有的，用模型知识补充，明确标注"（模型知识，vault 中暂无）"
- **诚实告知**：如果搜索后 vault 中确实没有相关内容，说"vault 中暂无此主题的记录"

回答格式：
```markdown
## 回答

{综合回答，穿插 [[文档名]] 引用}

---
📊 引用了 {N} 份文档：[[文档1]] [[文档2]]
💡 模型补充：{有/无}
```

### Step 5: 典型查询模式

#### 模式1：项目健康度查询

```
用户：PJ-001 现在怎么样？
```

搜索策略：
1. 读 `MOC-Projects.md` 确认 PJ-001 存在
2. 读 `00-项目章程.md`
3. 读 `04-监控/预算执行表.md`
4. 读 `04-监控/风险登记册.md`
5. 读最近的 `项目健康报告`

输出：综合健康度 + 关键指标 + 告警项

#### 模式2：预算超支项目筛选

```
用户：哪些项目预算超支了？
```

搜索策略：
1. 扫描所有 `项目库/*/04-监控/预算执行表.md`
2. 筛选 `CPI < 0.95` 或 `预算偏差率 > 5%`

输出：超支项目列表 + 超支金额 + 原因

#### 模式3：客户历史总览

```
用户：某市政府合作过哪些项目？
```

搜索策略：
1. 读 `客户档案/某市政府.md`
2. 搜索 `client: "某市政府"` 的所有项目章程

输出：项目列表 + 合同额汇总 + 合作评价

#### 模式4：投标成功率

```
用户：今年投标成功率多少？
```

搜索策略：
1. 扫描 `线索池/` 和 `投标档案/`
2. 统计：中标数 / 投标总数

输出：成功率 + 中标/未中标明细 + 趋势

#### 模式5：类似项目参考

```
用户：有没有做过类似的政务数字化项目？
```

搜索策略：
1. 搜索 `domain/政务` 或 `type/project`
2. 读匹配项目的章程和复盘

输出：类似项目列表 + 关键数据 + 经验教训引用

#### 模式6：风险预警

```
用户：哪些项目有高风险？
```

搜索策略：
1. 扫描所有 `04-监控/风险登记册.md`
2. 筛选 `risk_level: 高` 且 `status: open`

输出：高风险项目列表 + 风险描述 + 建议行动

## 与其他 Skill 的协作

| 场景 | 组合 |
|------|------|
| 查询后需更新 | `/query` → `/change` 或 `/monitor` |
| 查询后沉淀知识 | `/query` 发现 → `/meeting` 或 `/close` 复盘 |

---

## Notes

- `/query` 不直接修改 vault，只读
- 单次查询读取的文档控制在 10 份以内，避免 context 溢出
- 复杂查询（如跨项目统计）可能需要分步执行
- 好的查询结果可以通过其他 skill 回写（用户决定）
