# /monitor — 运营监控

项目治理的核心 skill。综合计算项目健康度，输出健康报告。

## 定位

| Skill | 职责 | 输出 |
|-------|------|------|
| `/plan` | 计划制定 | WBS + 里程碑 |
| `/meeting` | 会议纪要 | 会议文档 |
| `/change` | 变更管理 | 变更记录 |
| `/monitor` | **运营监控 — 项目健康度诊断** | 健康报告 + 工作台更新 |
| `/close` | 项目收尾 | 决算 + 复盘 |

**典型工作流**: 各运营 skill 写入后 → `/monitor` 定期扫描 → 生成健康报告

---

## Behavior

### Step 0: 确定范围

| action | 行为 |
|--------|------|
| `health`（默认）| 生成指定项目的健康报告 |
| `dashboard` | 更新 `个人工作台/今日关注.md` |
| `payments` | 回款跟踪汇总视图 |
| `cron` | 定时扫描所有活跃项目，静默更新工作台 |

### Step 1: 扫描项目文件

#### 1a. 选择项目

如 `project=all` 或未提供：

```bash
obsidian search query="type/project"
```

过滤 `status:active` 或 `status:planning` 或 `status:executing` 的项目。

#### 1b. 读取项目监控文件

对每个项目，读取：

```bash
obsidian read path="项目库/{project}/00-项目章程.md"
obsidian read path="项目库/{project}/02-计划/里程碑计划.md"
obsidian read path="项目库/{project}/04-监控/风险登记册.md"
obsidian read path="项目库/{project}/04-监控/预算执行表.md"
obsidian read path="项目库/{project}/04-监控/问题跟踪.md"
obsidian read path="项目库/{project}/03-执行/变更记录/"
```

### Step 2: 计算健康度

#### 2a. 进度健康度

```
进度健康度 = 里程碑达成率 × 关键路径偏差系数

里程碑达成率 = 已完成里程碑数 / 总里程碑数
关键路径偏差系数 = 1 - (逾期里程碑天数总和 / 项目总工期天数)
```

- 进度健康度 ≥ 0.9：🟢 健康
- 0.7 ≤ 进度健康度 < 0.9：🟡 预警
- 进度健康度 < 0.7：🔴 告警

#### 2b. 预算健康度

```
CPI = 已完成工作的预算价值 / 已完成工作的实际成本
SPI = 已完成工作的预算价值 / 计划工作的预算价值

预算健康度 = CPI × 0.7 + (1 - |预算偏差率|) × 0.3
预算偏差率 = (预测总成本 - 总预算) / 总预算
```

- CPI ≥ 0.95：🟢 健康
- 0.85 ≤ CPI < 0.95：🟡 预警
- CPI < 0.85：🔴 告警

#### 2c. 质量健康度

```
质量健康度 = 交付物一次通过率 × 0.6 + 问题闭环率 × 0.4

交付物一次通过率 = 一次评审通过的交付物 / 总交付物
问题闭环率 = 已关闭问题 / 总问题数
```

#### 2d. 风险健康度

```
风险健康度 = 1 - (高风险项数 × 0.3 + 中风险项数 × 0.1) / 总风险项数
```

- 高风险项 = 0：🟢 健康
- 1-2 个高风险项：🟡 预警
- ≥3 个高风险项：🔴 告警

#### 2e. 综合健康度

```
综合健康度 = 进度 × 0.3 + 预算 × 0.3 + 质量 × 0.2 + 风险 × 0.2
```

### Step 3: 生成健康报告

#### 3a. 单项目健康报告

文件路径：`项目库/{project}/04-监控/项目健康报告 YYYY-MM-DD.md`

```markdown
---
title: "项目健康报告 YYYY-MM-DD"
type: cost-report
project: "{project_id}"
report_date: YYYY-MM-DD
overall_health: 0.85
tags:
  - type/cost-report
---
```

## Notes

- `/monitor` 是项目治理的**核心诊断工具**
- 建议每周至少运行一次 `/monitor`
- cron 模式可设置每日自动扫描（见 cron 配置）
- 健康报告的"建议行动"是 AI 建议，最终决策由人拍板
