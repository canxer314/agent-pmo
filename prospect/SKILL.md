# /prospect — 线索管理

记录售前线索，跟踪客户接触过程。线索阶段的核心问题是"这个值不值得跟？"——收集信息、判断机会质量、决定跟不跟。一旦用户决定正式投入资源做方案，通过 `/bid action=new` 进入售前项目阶段。

## 定位

| Skill | 职责 | 输出 |
|-------|------|------|
| `/prospect` | 线索管理 — 记录和跟踪售前线索 | `线索池/` 下的线索文件夹（卡片 + raw/）|
| `/bid` | 投标管理 — 创建售前项目骨架，进入正式售前流程 | `售前项目/` 下的售前项目文件夹 |
| `/initiate` | 项目立项 — 中标后创建项目结构 | `项目库/` 下的项目文件夹 |

**典型工作流**: `/prospect` 创建线索 → 跟进收集 raw 素材 → 用户拍板 → `/bid action=new` 升级 → 售前项目 → `/initiate` 立项

---

## 线索状态机

```
新线索 → 跟进中 → 已转化（→ /bid action=new 进入售前）
              ↘ 已关闭（不跟）
```

| 状态 | 含义 | 触发条件 |
|------|------|----------|
| `新线索` | 刚创建，尚未开始跟进 | `/prospect action=new` 创建后 |
| `跟进中` | 正在收集信息、联系客户 | 用户首次更新跟进记录后 |
| `已转化` | 已升级为售前项目 | `/bid action=new` 创建售前项目后 |
| `已关闭` | 放弃跟进 | 用户确认关闭后 |

> **边界定义**：线索 → 售前的分界线是 `/bid action=new`。AI 在双提议中可以**建议**升级，但不能自动升级。

---

## Behavior

### Step 0: 确定动作

根据 `action` 参数：

| action | 行为 |
|--------|------|
| `new`（默认）| 创建新线索（文件夹 + 卡片 + raw/） |
| `update` | 更新现有线索（追加跟进记录、推进状态） |
| `list` | 列出所有活跃线索 |
| `dashboard` | 生成/更新 `线索看板.md` |
| `ingest` | 扫描 `inbox/`，建议匹配已有线索或创建新线索 |

---

### Step 1: 创建新线索（action=new）

#### 1a. 收集信息

与用户对话收集（未提供的字段会询问）：

| 字段 | 说明 | 示例 |
|------|------|------|
| 客户名称 | 客户全称 | 某市人民政府 |
| 需求主题 | 一句话描述需求 | 政务数字化平台升级 |
| 需求概述 | 客户要做什么、为什么做 | [详细描述] |
| 预计金额 | 预估合同额 | 500万 |
| 预计签约时间 | 客户预期或销售判断 | 2026-Q3 |
| 竞争对手 | 已知的竞争对手 | 某科技公司、某软件公司 |
| 关键干系人 | 客户方决策链 | 王主任（信息中心主任） |
| 信息来源 | 怎么知道这个线索的 | 老客户转介绍 |

#### 1b. 检查客户档案

```bash
obsidian search query="客户档案-{client}"
```

- 找到 → 记录客户档案路径，后续双提议时建议关联
- 未找到 → 双提议时建议新建客户档案

#### 1c. 准备线索文档内容

文件结构：
```
线索池/活跃线索/线索-{客户}-{主题}/
├── 线索-{客户}-{主题}.md          # 结构化卡片
└── raw/                            # 原始素材（聊天记录、PPT、Word等）
```

线索卡片内容：

```markdown
---
title: "线索-{客户}-{主题}"
type: prospect
status: 新线索 | 跟进中 | 已转化 | 已关闭
estimated_value: 5000000
expected_date: 2026-Q3
owner: 业务经理
client: "{客户}"
source: 老客户转介绍
competitors:
  - 某科技公司
  - 某软件公司
stakeholders:
  - 王主任（信息中心主任）
created: YYYY-MM-DD
updated: YYYY-MM-DD
tags:
  - type/prospect
  - status/新线索
---

# 线索：{客户} — {主题}

> 预计金额：500万 | 预计签约：2026-Q3 | 状态：新线索

## 需求概述

[客户要做什么、为什么做、什么时间要]

## 客户背景

[客户行业、规模、信息化现状]

## 竞争态势

| 竞争对手 | 已知优势 | 已知劣势 |
|----------|----------|----------|
| 某科技公司 | [优势] | [劣势] |

## 干系人

- 王主任（信息中心主任）— [角色描述]

## raw 素材索引

> 该线索的原始素材存放在 `raw/` 目录下。创建线索时如用户提供了素材，在此列出。

| 文件 | 日期 | 类型 | 说明 |
|------|------|------|------|
| [[raw/2026-05-10-王局聊天记录.txt]] | 05-10 | 聊天记录 | 王局长介绍项目背景 |

## 跟进记录

### YYYY-MM-DD

[本次沟通内容、客户反馈、下一步行动]

## 下一步

- [ ] 行动项1
- [ ] 行动项2
```

#### 1d. 双提议

```markdown
提议 1 — 关联建议：
1. [[客户档案-{客户}]] — 客户档案{已存在/不存在}，建议关联
2. [[知识库/经验教训/政务项目-经验教训]] — 历史有同类项目经验，建议关联

提议 2 — 后续行动：
1. 是否立即创建客户档案？（如不存在）
2. 是否有相关 raw 素材需要放入 raw/ 目录？（聊天记录、PPT、Word 等）
3. 📎 是否现在扫描 inbox/ 看是否有匹配的 raw 素材？

请回复编号确认，或"跳过"
```

#### 1e. 写入（如用户确认）

```bash
obsidian create path="线索池/活跃线索/线索-{客户}-{主题}/线索-{客户}-{主题}.md" content="..."
obsidian create path="线索池/活跃线索/线索-{客户}-{主题}/raw/.gitkeep" content=""
```

### Step 2: 更新现有线索（action=update）

#### 2a. 搜索线索

```bash
obsidian search query="type/prospect"
```

列出匹配线索，让用户选择：

```
找到以下活跃线索：
1. 线索-某市政府-数字化（跟进中，最后更新：05-10）
2. 线索-某国企-ERP升级（新线索，05-15创建）

请回复编号选择，或提供线索名称
```

#### 2b. 读取并追加

```bash
obsidian read path="线索池/活跃线索/线索-{客户}-{主题}/线索-{客户}-{主题}.md"
```

在文档末尾追加新的跟进记录：

```bash
obsidian append path="线索池/活跃线索/线索-{客户}-{主题}/线索-{客户}-{主题}.md" content="

### YYYY-MM-DD

[本次更新内容]

## 下一步

- [ ] 新行动项
"
```

更新 frontmatter 中的 `updated`。如状态变了，更新 `status` 和 `tags`。

#### 2c. 状态推进检查

如果首次从 `新线索` → `跟进中`：无额外动作，仅更新状态。

如果用户想 `跟进中` → `已转化`：
- AI 提示："线索升级为售前项目需要通过 `/bid action=new` 来完成。是否现在执行？"
- **AI 不能在此处自动 move 文件到已转化。** 升级的唯一通道是 `/bid action=new`。

如果 `跟进中` → `已关闭`：
- 提示用户："是否记录关闭原因？"
- 用户确认后，move 文件夹到 `线索池/已关闭/`：
  ```bash
  obsidian move path="线索池/活跃线索/线索-{客户}-{主题}" to="线索池/已关闭/线索-{客户}-{主题}"
  ```

### Step 5: 扫描 inbox 认领素材（action=ingest）

#### 5a. 扫描 inbox

```bash
obsidian files folder="线索池/inbox"
```

列出 inbox 中所有文件。

#### 5b. 逐个处理

对每个 raw 素材，尝试提取关键信息（客户名、需求关键词、来源人），搜索已有线索：

```bash
obsidian search query="type/prospect"
```

#### 5c. 输出建议

```markdown
📂 inbox 扫描 — 发现 {N} 份待认领素材

1. `2026-05-10-王局聊天记录.txt`
   → 提取：客户"某市政府"、主题"数字化"

2. `2026-05-15-某银行招标预告.pdf`
   → 提取：客户"某银行"、主题"风控系统"
   → 🔍 未找到匹配线索
   → 建议：创建新线索 `线索-某银行-风控系统`，或继续留在 inbox

请逐条确认操作
```

#### 5d. 执行

```bash
# 用户确认后：移动文件
cp "线索池/inbox/2026-05-10-王局聊天记录.txt" "线索池/活跃线索/线索-某市政府-数字化/raw/"
rm "线索池/inbox/2026-05-10-王局聊天记录.txt"

# 或：创建新线索
/prospect action=new client=某银行 topic=风控系统
```

## 与其他 Skill 的协作

| 场景 | 组合 |
|------|------|
| 线索升级为售前项目 | `/prospect` → `/bid action=new` |
| 线索中标，需立项 | `/prospect` → `/bid action=new` → `/bid action=result` → `/initiate` |
| 线索关闭，沉淀经验 | `/prospect` → 知识库经验教训 |

---

## Notes

- `/prospect` 是 vault 线索层的**唯一写入通道**
- 线索 → 售前的唯一分界线是 `/bid action=new`，AI 不可自动升级
- raw/ 目录的文件不改名、不要求 frontmatter，是收纳箱
- `线索看板.md` 由 `action=dashboard` 自动生成/更新
- `inbox/` 存放零散的、尚未关联线索的 raw 素材，由 `action=ingest` 认领
