# /payment — 回款跟踪

扫描所有活跃项目的合同付款节点，生成回款汇总视图。

## 定位

| Skill | 职责 | 输出 |
|-------|------|------|
| `/initiate` | 项目立项 — 合同关键条款 | 付款节点定义 |
| `/monitor` | 运营监控 — 个人工作台更新 | 今日关注中的回款提醒 |
| `/payment` | **回款跟踪 — 付款节点汇总与催款** | 回款跟踪文档 + 汇总视图 |
| `/close` | 项目收尾 — 决算 | 决算报告 |

**典型工作流**: `/initiate` 定义付款节点 → `/payment` 日常跟踪 → `/close` 决算核对

---

## Behavior

### Step 0: 确定范围

如 `project` 参数已提供，仅扫描指定项目。否则扫描所有活跃项目：

```bash
obsidian search query="type/project"
```

过滤 `status:active` 或 `status:executing` 的项目。

### Step 1: 读取合同付款节点

对每个项目读取：

```bash
obsidian read path="项目库/{project}/01-合同/主合同关键条款.md"
obsidian read path="项目库/{project}/00-项目章程.md"
```

提取付款节点信息：
- 合同金额
- 付款节点（比例、金额、条件）
- 计划收款日期（如果有记录）

**方案B：更新回款跟踪文档**

读取 `项目库/{project}/04-监控/回款跟踪.md`（如存在），或提示用户创建。

### Step 3: 回款跟踪文档格式

```markdown
---
title: "payment-tracking"
type: cost-report
project: "{project_id}"
contract_value: 4800000
tags:
  - type/cost-report
---

# 回款跟踪 — {项目名}

## 合同付款节点

| 节点 | 比例 | 金额 | 计划收款日 | 实际收款日 | 状态 |

| 第二笔 | 30% | 144万 | 2026-06-15 | | ⬜ 待收款 |

- [x] 首款 144万 — 2026-05-10 已收
- [ ] 第二笔 144万 — 计划 2026-06-15

- 2026-05-10：首款已收，客户很满意
- 2026-06-01：第二笔款到期前已发催款函
```

### Step 4: 写入回款跟踪文档

如用户选择方案B：

```bash
obsidian create path="项目库/{project}/04-监控/回款跟踪.md" content="..." overwrite
```

### Step 5: 双提议

```markdown
✓ 确定性链接已自动建立：
- [[01-合同/主合同关键条款]] — 付款节点来源（同项目固定文件名）

提议：
1. 是否同步更新「个人工作台/今日关注.md」中的回款提醒？
2. 逾期款项是否需要记录催款动作？
3. 是否需要通知业务经理跟进？

请回复编号确认，或"跳过"
```

### Step 6: 输出

展示汇总视图（方案A）。如需创建/更新回款跟踪文档（方案B），确认后执行。

## 与其他 Skill 的协作

| 场景 | 组合 |
|------|------|
| 回款节点来自合同 | `/initiate` → `/payment` |
| 回款提醒同步工作台 | `/payment` → `/monitor action=dashboard` |
| 决算核对回款 | `/payment` → `/close` |

---

## Notes

- `/payment` 扫描所有活跃项目的合同付款节点
- 方案A（对话中展示）是推荐的默认输出方式
- 方案B（写入文档）适合需要长期跟踪回款状态的场景
- 回款跟踪文档（`04-监控/回款跟踪.md`）由 `/initiate` 创建框架，`/payment` 更新
- 逾期款项应在双提议中建议记录催款动作
